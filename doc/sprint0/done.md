**Definition of Done**

A user story or feature is considered done when it meets all of the following criteria:

**1. Code Complete**

* The feature has been fully implemented and merged into the main branch without conflicts.


**2. Functionality Verified**


* The feature works as described in the user story and passes all manual and unit tests performed by the team.


**3. No Major Bugs**


* There are no known critical issues or crashes that affect core functionality.


**4. Code Quality and Documentation**


* Code follows team conventions for naming and formatting.


* Important classes, methods, or logic are commented, and documentation is updated where needed.


**5. Integrated with the Project**


* The feature interacts correctly with existing components and does not break previously working features.


**6. Committed and Tracked**


* All relevant files are committed and pushed to GitHub.


* The corresponding Trello or project management card is updated to show completion.


**7. Team Review and Approval**


* The team reviews the work during the sprint review and agrees it meets all acceptance criteria.
